# Contributing to Paula Portfolio

## Prerequisites

- A modern web browser (Chrome, Firefox, Safari, Edge)
- [Python 3.x](https://www.python.org/) (for local development server)
- [Azure CLI](https://docs.microsoft.com/cli/azure/install-azure-cli) (for image uploads and infrastructure)
- [Git](https://git-scm.com/)

## Local Development

1. **Clone the repository**
   ```bash
   git clone https://github.com/juancamilor/paula-portfolio.git
   cd paula-portfolio
   ```

2. **Start a local server**
   ```bash
   python -m http.server 8080
   ```

3. **Open in browser**
   ```
   http://localhost:8080
   ```

> **Note:** Images are hosted on Azure Blob Storage, so you need an internet connection for images to load during local development.

## Making Changes

1. Create a feature branch:
   ```bash
   git checkout -b feature/my-change
   ```

2. Make your changes to HTML, CSS, or JS files.

3. Test locally with `python -m http.server 8080`.

4. Commit and push:
   ```bash
   git add .
   git commit -m "Description of changes"
   git push origin feature/my-change
   ```

5. Open a Pull Request on GitHub. The CI/CD pipeline will automatically create a staging preview.

6. Once merged to `main`, the site deploys automatically.

## Managing Images

Images are stored in **Azure Blob Storage**, not in the Git repository.

### Upload a new image
```bash
az storage blob upload \
  --account-name stpaulaportfolio \
  --container-name images \
  --file ./my-photo.jpg \
  --name my-photo.jpg
```

### List all images
```bash
az storage blob list \
  --account-name stpaulaportfolio \
  --container-name images \
  --output table
```

### Delete an image
```bash
az storage blob delete \
  --account-name stpaulaportfolio \
  --container-name images \
  --name old-photo.jpg
```

### Reference in HTML
```html
<img src="https://stpaulaportfolio.blob.core.windows.net/images/my-photo.jpg" alt="Description">
```

You can also manage images via the [Azure Portal](https://portal.azure.com) or [Azure Storage Explorer](https://azure.microsoft.com/products/storage/storage-explorer/).

## Deployment

Deployment is fully automated:

- **Push to `main`** → GitHub Actions deploys to production
- **Pull Requests** → GitHub Actions creates a staging preview environment
- **No build step** required — the site is pure HTML/CSS/JS

## Infrastructure

To recreate the Azure resources from scratch, run:
```powershell
.\infra\setup.ps1
```

This provisions the resource group, storage account, and blob container. The Azure Static Web App must be created separately via the Azure Portal or CLI (it links to the GitHub repo).
